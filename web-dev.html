The Internet & Web Development

Study Notes

Computers connected to the internet are called clients and servers. Clients = your devices (i.e. laptop via wifi, cellphone via mobile network) Servers = computers that store all the info you access with your 'client' device (i.e. websites and Apps).

Requirements for clients and servers to 'communicate' with each other.

  • Internet Connection
  • TCP/IP = Transmission Control Protocol/Internet Protocol = Protocol that allows your device/internet connection to safely send data/packets to request/receive the information you requested.
  • DNS = Domain Name System = A website address is an IP address (on which server the information lives), to make it easier for us mere humans to remember, there is a DNS (address book) that keeps record of the IP address vs the domain name. This allows you to request data from the domain name instead of remembering/inputting the IP address for the webpages or site you are looking for.
  • HTTP = Hyper Text Transfer Protocol = set of rules that devices use to communicate with each other over the internet in order to send and receive data.
  • Component Files = The different files that make up a website. There are two types - Code Files (i.e. HTML & CSS) and Asssets (i.e. Images, music, videos,.. Media resources.)

Process flow for clients and servers to communicate with each other :

Browser requests website/data → DNS finds the correct address where the data lives → Browser (using TCP/IP) sends and HTTP request to the server requesting the data you want shown on your desktop/device → server returns requested data in small packets → browser reassembles data for your view on your monitor.

Component file parsing order = HTML (checks for additional links), then parses these links such as CSS and JS depending on the structure of the HTML instructions.

  • HTML = Structures website content
  • CSS = Applies styling to websites
  • JS = Adds interactivity to websites
  • SQL = ALlows your web application to stoore and retrieve data
